# 移除视口优化模式提示设计

## 背景

编辑器进入 `viewport-optimized` 模式时，右下角会显示“视口优化模式”浮层。该模式只延迟加载接近可见区域的图片和代码块，正文编辑能力不变；持续提示会增加用户认知和操作负担。

## 目标与范围

- 编辑器进入 `viewport-optimized` 模式时，不创建或显示运行时提示浮层。
- 保留 `viewport-optimized` 的性能优化、文档运行时标记和模式切换逻辑。
- 保留 `lightweight-edit` 的提示及“尝试完整模式”操作，因为该模式会暂停部分编辑器能力。
- 不修改大文档编辑器页面内已有的模式说明，不扩展到其他提示或性能策略。

## 实现方案

将运行时浮层的可见条件从同时覆盖 `viewport-optimized` 和 `lightweight-edit`，收窄为仅覆盖 `lightweight-edit`。删除不再使用的视口优化浮层文案，但保留相关 CSS 性能规则。

## 验证

- 增加回归测试：设置 `viewport-optimized` 决策后，运行时提示不存在或保持隐藏。
- 保留并运行现有测试：`lightweight-edit` 仍显示提示，点击操作后恢复普通模式并隐藏提示。
- 运行相关 Vitest；随后运行前端构建，确认类型检查和生产构建通过。
