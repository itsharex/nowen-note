# Tiptap 顶层表格 Block Patch 设计

## 范围

本阶段只支持替换一个已经存在的顶层 `table` 节点。表格被视为原子 Block：保留表格自身的 `blockId`，不提供行、单元格或单元格内部节点的独立 Patch，也不开放表格 `create`。

## 数据契约

- `table` 纳入富文本 Block 身份和索引模型，由编辑器、转换器和后端归一化统一保留 `blockId`。
- `replace.node` 使用明确的表格节点白名单：`table -> tableRow -> tableCell/tableHeader -> paragraph -> text/hardBreak`。
- 表格、行、单元格和段落属性均使用白名单；链接和文本样式继续复用现有安全校验。
- 单个替换节点仍限制为 256 KB，并限制行数、单行单元格数和单元格总数。
- 表格替换只能命中顶层同类型 `table`；任何嵌套表格、类型转换、未知节点或未知属性均拒绝。
- 替换节点内所有 `blockId` 必须合法且互不重复；除目标表格 ID 外，不得与文档中替换范围外的 Block ID 冲突。

## 保存与索引

前端规划器仅在前后文档拥有相同顶层表格 ID 时生成一个整表 `replace`，随后仍以完整 JSON 回放相等作为 fail-closed 条件。后端在同一数据库事务中写入正文、版本历史和 Block 索引；表格替换走完整索引同步，不使用叶子增量索引。

## 回退

新增/删除表格、表格移动、表格与其他节点互转、超限表格、复杂单元格内容或回放不一致时，规划器返回 `null`，由现有整篇保存路径处理。
