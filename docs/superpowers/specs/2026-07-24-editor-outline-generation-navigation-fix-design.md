# 编辑器大纲生成与跳转修复设计

## 问题

- 富文本优化运行模式会主动停止向 `EditorPane` 发布大纲。
- 窗口化富文本只接收第一章节的大纲，并让每个章节分别注册跳转回调，后挂载章节会覆盖前一章节。
- 大 Markdown 的预览模式仍把大纲跳转发送给隐藏的 CodeMirror 源码编辑器。

## 设计

1. 优化模式仍通过既有 Worker 分析链路发布大纲；优化只限制重节点和实时装饰，不再关闭大纲导航。
2. 窗口化富文本由外层统一生成全部章节的大纲和导航令牌。点击令牌后先挂载目标章节，再调用该章节编辑器的本地跳转函数；子编辑器不再直接覆盖父级回调。
3. 大 Markdown 同时保存源码编辑器与预览根节点引用。源码模式滚动 CodeMirror，预览模式滚动渲染后的标题。
4. 不改变普通文档的数据结构、保存协议或大纲 UI，仅修正运行时接线。

## 验证

- 优化模式仍向富文本编辑器传递大纲回调。
- 窗口化富文本大纲包含所有章节，点击后挂载并定位正确章节。
- 大 Markdown 预览点击大纲只滚动可见预览。
- 相关定向测试、TypeScript 检查和生产构建通过。
