# 同步版本冲突处理设计

## 背景

当前同步状态只告诉用户存在若干条版本冲突，并提示“待人工处理”，但没有说明是哪篇笔记，也没有提供可执行的处理入口。同步队列还会直接展示 UUID、英文错误和不可点击的“重试”按钮，导致用户既无法理解问题，也无法完成处理。

本设计将冲突处理收敛为一个完整闭环：识别冲突笔记、查看两个版本、明确选择保留版本、保证另一版本仍可找回，并确认处理结果。

## 目标

- 用户能通过笔记标题和所属笔记本识别每一条冲突。
- 用户能在选择前预览此设备版本和服务器版本。
- 用户能直接完成“保留此设备版本”或“使用服务器版本”。
- 无论选择哪一份，另一份内容都不会不可恢复地丢失。
- 全部冲突处理完成后，同步状态恢复为正常状态。

## 非目标

- 不提供逐行 Diff、三方合并或可编辑的合并器。
- 不自动判断哪个版本更正确。
- 不改变普通网络错误和其他可重试队列项的处理方式。
- 不重构无关的同步、编辑器或版本历史代码。

## 信息架构

### 数据管理页同步卡片

存在版本冲突时，卡片显示：

- 主信息：`发现 5 篇笔记存在版本冲突，两份内容均已保留。`
- 主操作：`处理 5 篇冲突`
- 次操作：`刷新服务器内容`
- 上次同步时间保持可见。

“刷新服务器内容”只更新服务器快照，不再使用主按钮样式，也不暗示它能够解决冲突。

### 全局同步队列

冲突条目默认展示：

- 笔记标题；标题无法取得时显示“未命名笔记”。
- 所属笔记本；无法取得时显示“未知笔记本”。
- 此设备修改时间。
- 一行本地内容摘要。
- `版本冲突`状态标签。
- `处理冲突`按钮。

UUID、HTTP 状态、错误码和重试次数放入可展开的“技术详情”，不作为主信息。非冲突项继续使用现有重试流程。

### 冲突处理中心

从数据管理页进入时，打开包含全部冲突笔记的处理中心；从同步队列单条进入时，直接选中对应笔记。

左侧或顶部列表显示待处理笔记。主区域显示当前笔记的双版本预览：

- `此设备版本`：读取冲突队列保存的本地 payload。
- `服务器版本`：进入处理页时按 noteId 获取服务器最新内容。
- 两侧均显示标题、更新时间、内容格式和只读正文预览。
- 标题或正文不同时给予轻量差异提示，但不提供逐行合并。

界面顶部固定提示：`两份内容都已保留，请查看后选择最终版本。`

## 处理动作

### 保留此设备版本

1. 用户点击 `保留此设备版本`。
2. 二次确认说明：此设备内容将成为最新内容，服务器当前内容仍可从版本历史恢复。
3. 使用已获取的服务器最新 version 提交本地 payload，避免盲目重放旧版本号。
4. 后端按现有笔记更新流程将服务器当前内容写入版本历史。
5. 服务端确认成功后才移除对应冲突队列项，并刷新笔记与同步摘要。

若提交期间服务器版本再次变化，保留冲突项并重新加载两份内容，提示用户重新确认，不自动覆盖。

### 使用服务器版本

1. 用户点击 `使用服务器版本`。
2. 二次确认说明：当前服务器内容将继续作为正式笔记，本地内容会保存为可见的冲突副本。
3. 先在原笔记所属笔记本创建一篇新笔记，标题为 `原标题（冲突副本 YYYY-MM-DD HH:mm）`，内容和格式取自本地 payload。
4. 副本获得服务器 ACK 后，才移除对应冲突队列项。
5. 刷新原笔记、笔记列表与同步摘要，并提供 `打开冲突副本`入口。

如果副本创建失败，不移除冲突项，也不切换到服务器版本，提示用户稍后重试或导出本地副本。

### 稍后处理

关闭处理界面，不修改服务器内容、本地 payload 或冲突队列。

## 数据与组件边界

- `offlineQueue` 继续负责持久保存冲突 payload，并新增仅针对明确处理结果的安全移除或状态更新能力。
- 冲突解析服务负责把队列项与服务器笔记、笔记本信息组合成 UI 可用模型，并执行两种解决动作。
- `ConflictResolutionCenter` 负责列表、双版本预览、确认和处理状态，不直接拼装底层请求。
- `OfflineIndicator` 和 `DataManager` 只负责展示摘要并打开处理中心。
- 现有 `VersionHistoryPanel` 继续承担处理后的历史恢复，不把冲突处理逻辑塞入版本历史面板。

## 状态与错误处理

- 加载服务器版本时显示骨架屏；单条加载失败时保留本地预览并提供重试。
- 笔记已被服务器删除时，服务器侧明确显示“服务器版本已删除”，仍允许将本地内容恢复为新笔记。
- 处理按钮执行期间禁用重复提交。
- 只有服务器确认目标动作完成后才能删除冲突队列项。
- 关闭弹窗、刷新页面或应用退出均不得丢失未处理冲突。
- 全部冲突解决后显示：`所有版本冲突已处理，同步已恢复。`

## 可访问性与响应式

- 桌面端双栏并排，窄屏改为“此设备 / 服务器”分段切换。
- 主操作名称直接描述结果，不使用“确认”“继续”等模糊按钮文案。
- 状态不只依赖颜色，同时使用文字和图标。
- 对话框支持焦点圈定、Esc 关闭和键盘操作；二次确认默认聚焦取消按钮。

## 验收标准

1. 有 5 篇冲突时，数据管理页显示“5 篇笔记”并提供明确的处理入口。
2. 每条冲突默认显示标题而非 UUID，且能定位到所属笔记本。
3. 用户选择前能查看本地和服务器的标题、时间及正文预览。
4. “保留此设备版本”成功后，服务器旧内容可从版本历史恢复。
5. “使用服务器版本”成功后，本地内容作为独立冲突副本可见。
6. 任一步骤失败时，冲突队列项仍存在，本地 payload 不丢失。
7. 服务器在确认期间再次变化时不会被自动覆盖。
8. 单条处理完成后计数立即减少；全部完成后冲突提示消失并显示成功反馈。
9. 普通离线重试、非冲突失败项和现有版本历史功能不回归。

## 测试范围

- 单元测试：冲突 UI 模型映射、标题降级、冲突计数、队列安全移除、二次 409 保留。
- 组件测试：列表识别、双版本加载、两种确认文案、加载和错误状态、窄屏切换。
- 集成测试：保留本地后版本历史可恢复；使用服务器后冲突副本成功创建；副本失败时队列不删除。
- 回归测试：非冲突队列仍可单条或批量重试，刷新服务器内容不改变冲突项。
