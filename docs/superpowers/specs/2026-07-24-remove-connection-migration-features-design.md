# 移除“连接与账号”和“一次性数据迁移”功能设计

## 目标

从客户端和服务端完整移除多服务器“连接与账号”及“一次性数据迁移”两条功能链，不保留隐藏入口、可调用接口或专用凭据。普通登录、当前服务器地址、数据备份与导入导出、数据库结构迁移继续保留。

## 保留边界

- 保留登录页的服务器地址输入、连通性检测、用户名密码登录和现有自动登录能力。
- 保留当前会话使用的服务器 URL、登录令牌及普通登录凭据。
- 保留数据管理中的备份、恢复、导入、导出和本地数据目录迁移。
- 保留后端数据库版本升级、管理员用户归属调整及其他与本功能无关的数据转移能力。
- 不删除用户笔记、附件、任务、版本历史或已有数据库内容。

## 前端调整

- 删除账号弹层中的“连接与账号”和“迁移数据”入口。
- 删除用于打开连接中心的桌面端“连接 NAS / 云端”入口；已经连接远程服务器时仍保留现有“切回本地离线模式”能力。
- 删除全局 `ServerConnectionCenter` 挂载、弹窗组件及其事件协议。
- 删除多服务器资料、专用凭据保险箱和迁移 V2 客户端模块，以及对应的专用测试。
- 简化登录页，移除“待重新认证服务器资料”的预填和回写逻辑；手动填写服务器地址的登录流程不变。
- 账号弹层只展示当前实际服务器和现有账号操作，不再订阅或显示多服务器资料状态。

## 桌面端与 Android 凭据清理

- Electron 的普通登录凭据读写继续保留；删除多服务器 profile 的 preload API、IPC handler 和业务方法。
- 升级后首次启动时清空 `credentials.json` 中旧的 `profiles` 字段，同时保留其中的普通 `remember` 登录记录。
- 清理浏览器存储中的多服务器资料、活动资料、旧云端登录记录、凭据索引和待重新认证标记。
- Android 根据旧凭据索引删除 `serverAccount.<profileId>.v1` 安全存储项；清理完成后再删除索引。
- 清理逻辑只处理已知的专用键，不删除 `nowen-server-url`、当前令牌或其他登录与业务存储。

## 后端调整

- 取消 `/api/user-migration` 和 `/api/user-migration/v2` 的路由注册。
- 删除 V1、V2 迁移路由及其注册副作用模块和专用测试。
- 移除 CORS 策略对迁移 V2 注册模块的副作用导入。
- 删除后访问上述路径应返回 404，不提供兼容转发或隐藏开关。
- 不删除被其他业务使用的通用仓储方法，仅删除一次性迁移独占的路由和代码。

## 兼容性与失败处理

- 历史专用存储清理采用 fail-safe：单个安全存储项删除失败时记录警告，不阻止应用启动；索引保留以便下次重试。
- Electron 旧 profile 清理不得破坏普通记住密码记录，需用测试覆盖两者共存的升级场景。
- 删除入口后不新增替代提示；用户仍可在登录页直接指定服务器。

## 验证

- 静态检索确认“连接与账号”“迁移数据”“连接 NAS / 云端”入口、连接中心事件和 `/api/user-migration` 客户端调用均已消失。
- 前端测试覆盖账号弹层入口删除、登录流程不依赖 profile，以及 Web/Android 专用存储清理边界。
- Electron 测试覆盖删除 profiles 且保留普通 remember 凭据，并确认 profile IPC 不再暴露。
- 后端测试确认迁移路由未注册且相关路径返回 404。
- 运行前端定向测试与生产构建、后端定向测试和类型检查；如全量测试存在既有失败，单独记录基线与本次无新增失败的证据。
