# 第三方图床退役与历史图片迁移

Nowen Note 已停止把新图片直接上传到第三方图床。所有新图片都会先创建为 Nowen 附件，再由附件存储驱动决定文件实际保存在：

- 本地附件目录；
- AWS S3；
- Cloudflare R2；
- MinIO；
- 其他 S3 兼容对象存储。

这样图片始终拥有附件 ID、笔记归属、权限、哈希、备份、导出和迁移关系。

## 对已有数据的影响

升级后不会主动修改已有笔记，也不会删除第三方 Bucket 中的图片。

历史正文中的公开图片 URL 会继续按原地址渲染。只要原 Bucket、域名和对象仍可访问，图片不会因为升级立即失效。

后端 `/api/image-hosting/upload` 已返回 `410 IMAGE_HOSTING_RETIRED`，旧客户端也不能再产生新的第三方图床链接。

## 迁移历史图片

系统管理员可以打开：

```text
设置 → 第三方图床
```

该页面现在是临时的历史图片迁移面板，而不是图床配置页。

迁移步骤：

1. 切换到需要处理的个人空间或工作区；
2. 确认旧图床图片 URL 前缀；
3. 扫描当前空间；
4. 查看受影响笔记和图片数量；
5. 执行迁移；
6. 检查失败项目并重试；
7. 所有空间迁移完成后，删除旧图床配置。

迁移会逐张执行：

```text
读取旧 URL
→ 服务端安全下载
→ MIME 与大小校验
→ SHA-256 去重
→ 创建 Nowen 附件
→ 替换笔记正文 URL
→ 使用笔记版本号安全保存
```

下载过程复用 Nowen 的远程图片导入安全策略，包括 HTTP/HTTPS 限制、SSRF 防护、超时、大小限制和内容类型嗅探。

## 失败与停止

- 单张图片失败时保留原 URL，不会写入损坏链接；
- 笔记版本冲突时自动重新读取并重试一次；
- 用户可以在完成当前图片后停止；
- 已完成的笔记不会因停止而回滚；
- 导入成功但正文保存失败产生的临时附件，可由附件孤儿清理流程处理。

## 删除旧配置

“删除旧配置”只删除 Nowen 数据库中的：

- 图床 Endpoint；
- Bucket 名称；
- Access Key 标识；
- 加密后的 Secret；
- Public Base URL；
- 历史失败回退策略。

它不会访问、删除或修改第三方 Bucket 中的任何对象。

建议迁移完成后保留 Bucket 备份一段时间，确认多端、分享页、导出文件和历史版本均无异常后，再由管理员自行清理第三方存储。

## 后续图片存储

需要使用 S3、R2 或 MinIO 时，请改用：

```text
设置 → 数据管理 → 附件存储
```

附件对象存储与旧图床的关键区别是：正文保存 Nowen 附件地址，文件存储位置由服务端驱动透明处理，而不是把第三方公开 URL 直接写入笔记。
